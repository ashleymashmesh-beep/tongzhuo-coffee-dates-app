# 同桌 App - 阿里云迁移指南

## 📋 迁移概述

本文档说明如何将同桌 App 从 Vercel + Firebase Auth 迁移到阿里云 OSS + 阿里云短信服务。

---

## 🔧 已完成的代码变更

### 1. 前端代码修改

| 文件 | 变更内容 |
|------|----------|
| `src/pages/Login.jsx` | 移除 Firebase Auth，改为调用阿里云 FC API |
| `src/contexts/AuthContext.jsx` | 使用自定义 token 验证，手机号作为用户 ID |
| `src/pages/Profile.jsx` | 移除 Firebase Auth 导入，使用 context logout |
| `src/lib/firebase.js` | 移除 Auth，只保留 Firestore 和 Storage |
| `.env.example` | 更新环境变量模板 |

### 2. 阿里云函数计算代码

新建 `aliyun-fc/send-sms/` 目录：
- `index.js` - 函数计算处理逻辑
- `package.json` - 依赖配置
- `template.yml` - 函数计算配置模板

## 🚀 部署步骤

### 第一步：开通阿里云短信服务

1. 登录 [阿里云控制台](https://console.aliyun.com/)
2. 开通「短信服务」
3. 添加签名（需要审核）：
   - 名称：同桌
   - 适用场景：验证码
4. 添加模板（需要审核）：
   - 类型：验证码
   - 内容：`您的验证码是${code}，5分钟内有效。`
5. 记录模板代码（如：SMS_123456789）

### 第二步：创建 AccessKey

1. 点击右上角头像 → AccessKey 管理
2. 创建 AccessKey
3. 保存 `AccessKey ID` 和 `AccessKey Secret`

### 第三步：部署函数计算

#### 方式 A：通过控制台部署（推荐）

1. 进入 [函数计算控制台](https://fc.console.aliyun.com/)
2. 创建服务：`tongzhuo-service`
3. 创建函数：`send-sms`
4. 粘贴 `aliyun-fc/send-sms/index.js` 代码
5. 配置环境变量：

   | 变量名 | 值 |
   |--------|-----|
   | ACCESS_KEY_ID | 你的 AccessKey ID |
   | ACCESS_KEY_SECRET | 你的 AccessKey Secret |
   | SIGN_NAME | 同桌 |
   | TEMPLATE_CODE | 你的短信模板代码 |

6. 添加依赖（package.json）：
   ```json
   {
     "dependencies": {
       "@alicloud/dysmsapi20170525": "^3.0.0",
       "@alicloud/openapi-client": "^0.4.6"
     }
   }
   ```

7. 配置 HTTP 触发器：
   - 路径：`/sms`
   - 方法：POST
   - 认证：匿名

8. 记录函数 URL（如：`https://xxx.cn-shanghai.fc.aliyuncs.com/sms`）

#### 方式 B：通过阿里云 CLI 部署

```bash
# 安装阿里云 CLI
# 然后执行：
cd aliyun-fc/send-sms
aliyun fc deploy --template-file template.yml
```

### 第五步：部署到阿里云 OSS

#### 方式 A：通过 OSS 控制台

1. 进入 [OSS 控制台](https://oss.console.aliyun.com/)
2. 创建 Bucket：
   - 名称：tongzhuo-app
   - 地域：选择就近区域
   - 访问权限：公共读
3. 开通静态网站托管
4. 上传 `dist` 目录内容

#### 方式 B：使用阿里云 CLI

```bash
# 构建前端
npm run build

# 上传到 OSS
ossutil cp -r dist/ oss://tongzhuo-app/ --update
```

## ⚠️ 注意事项

### 安全问题

1. **保护 AccessKey Secret**
   - 不要提交到代码仓库
   - 不要在前端代码中使用

2. **函数计算 URL 保护**
   - 当前使用匿名访问
   - 生产环境建议添加签名验证

### 数据迁移

**用户数据迁移**（Firebase Auth → 自定义认证）：

Firestore 中的用户数据结构变化：
```javascript
// 旧：使用 Firebase UID
{
  id: "firebase_uid_here",
  phoneNumber: "+8613800138000",
  ...
}

// 新：使用手机号作为 ID
{
  id: "13800138000",  // 直接使用手机号
  phoneNumber: "13800138000",
  ...
}
```

如果需要迁移现有用户，需要编写数据迁移脚本。
